# lunarbase-pmm-math-node

N-API binding that exposes the [`lunarbase-pmm-math`](../../rust/lunarbase-pmm-math)
Rust core to Node.js. The build produces a platform-specific native addon
(`.node`) plus a thin JS/TS wrapper.

## Requirements

- Node.js **>= 18**
- Rust toolchain (stable, `>= 1.77`) — install via [rustup](https://rustup.rs)
- A C toolchain matching your platform:
  - macOS: Xcode Command Line Tools (`xcode-select --install`)
  - Linux (glibc): `build-essential`
  - Linux (musl/Alpine): `build-base` + see [musl note](#musl--alpine) below
  - Windows: MSVC Build Tools

The build orchestrator is [`@napi-rs/cli`](https://napi.rs), pinned in
`devDependencies`.

## Build

From this directory:

```bash
npm install
npm run build          # release build
npm run build:debug    # unoptimized, faster to compile
```

What `npm run build` does:

1. invokes `napi build --platform --release`, which runs `cargo build` for the
   current host triple and emits a `.node` file with the host triple appended
   to the name (e.g. `lunarbase-pmm-math-node.darwin-arm64.node`);
2. renames the auto-generated CJS loader from `index.js` to `index.cjs` (so the
   ESM `wrapper.js` can `require()` it without conflicting with `"type":
   "module"` in `package.json`);
3. regenerates `index.d.ts` from the `#[napi]` annotations in `src/lib.rs`.

Artifacts that end up in this directory after a build:

| File                                            | Purpose                                            |
| ----------------------------------------------- | -------------------------------------------------- |
| `lunarbase-pmm-math-node.<triple>.node`         | Native addon, host-specific. Loaded at runtime.    |
| `index.cjs`                                     | CJS loader generated by napi-rs. Picks the right `.node` per platform. |
| `index.d.ts`                                    | TypeScript types, regenerated on every build.      |
| `wrapper.js`                                    | ESM entry that re-exports from `index.cjs`.        |

The `files` field in `package.json` controls what gets shipped on
`npm publish` — only `wrapper.js`, `index.cjs`, `index.d.ts`, and the
`*.node` artifacts.

### Cross-compilation

`@napi-rs/cli` supports cross-compiling to other triples via `--target`:

```bash
npx napi build --platform --release --target x86_64-unknown-linux-gnu
npx napi build --platform --release --target aarch64-apple-darwin
```

You need the matching Rust target (`rustup target add <triple>`) and a
cross-linker. For multi-platform npm releases, `napi prepublish` (already
wired into `prepublishOnly`) packages the per-triple `.node` files into
sub-packages.

### musl / Alpine

musl statically links libc by default, which prevents producing a `cdylib`.
To build on Alpine, disable `crt-static`:

```bash
RUSTFLAGS="-C target-feature=-crt-static" npm run build
```

This matches how Node.js binaries on Alpine and napi-rs musl addons ship.

## Test

```bash
npm test
```

Runs `node --test` against `test/quote.test.mjs`, which validates the binding
against the JSONL vectors generated from the on-chain Solidity reference
(`deterministic_vectors.jsonl`, `fuzz_vectors.jsonl` in the Rust crate
directory).

## Use from TypeScript

After `npm run build` the package is consumable locally. There are two
typical setups.

### 1. As a workspace / local dependency

In your consumer `package.json`:

```json
{
  "dependencies": {
    "lunarbase-pmm-math-node": "file:../path/to/math/rust-node/lunarbase-pmm-math-node"
  }
}
```

Then in TypeScript (works under both ESM and CJS — `package.json` exports
both `import` and `require` conditions):

```ts
import { quoteXToY, quoteYToX, type QuoteParams, type QuoteResult } from "lunarbase-pmm-math-node";

const params: QuoteParams = {
  sqrtPriceX48: "12781077694964135",
  feeQ48:       "834787359210",
  reserveX:     "100000000000000000000",
  reserveY:     "196452000000000000000000",
  concentrationK: 5000,
  amountIn:     "1000000",
};

const out: QuoteResult = quoteXToY(params);
console.log(out.amountOut, out.sqrtPriceNext, out.fee);
```

All numeric inputs are passed as **strings** — decimal or `0x`-prefixed hex —
to preserve full `U256` precision. Outputs are decimal strings.

### tsconfig

For the imports above to type-check you need `"moduleResolution": "node16"`
(or `"bundler"`) and `"esModuleInterop": true`. A minimal setup:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ES2022",
    "moduleResolution": "node16",
    "esModuleInterop": true,
    "strict": true
  }
}
```

### 2. From a sibling directory without publishing

If you don't want to bother with `file:` deps, you can import the wrapper
directly by path:

```ts
// ESM
import { quoteXToY } from "../math/rust-node/lunarbase-pmm-math-node/wrapper.js";
```

```ts
// CJS
const { quoteXToY } = require("../math/rust-node/lunarbase-pmm-math-node/index.cjs");
```

## API

```ts
interface QuoteParams {
  sqrtPriceX48: string;          // decimal or 0x-hex, U256
  anchorSqrtPriceX48?: string;   // defaults to sqrtPriceX48
  feeQ48: string;                // U256, fee in Q48
  reserveX: string;              // U256
  reserveY: string;              // U256
  concentrationK: number;        // multiplier k
  amountIn: string;              // U256
}

interface QuoteResult {
  amountOut: string;       // U256, decimal
  sqrtPriceNext: string;   // U256, decimal
  fee: string;             // U256, decimal
}

function quoteXToY(params: QuoteParams): QuoteResult;
function quoteYToX(params: QuoteParams): QuoteResult;
```

The math is bit-exact with the on-chain Solidity reference; see the
[root README](../../../README.md) for the full picture.

## Troubleshooting

- **`Error: Cannot find module './lunarbase-pmm-math-node.<triple>.node'`** —
  the addon for your platform isn't built. Run `npm run build` again or
  install on a supported triple.
- **`undefined symbol` / linker errors on Alpine** — see
  [musl / Alpine](#musl--alpine).
- **`index.js` reappears after a build** — expected; the `build` script
  renames it to `index.cjs`. If you ran `napi build` directly, do the
  rename yourself or just run `npm run build`.
- **TypeScript can't find the module** — make sure `moduleResolution` is
  `node16` or `bundler` so the `exports` map in `package.json` is honored.
